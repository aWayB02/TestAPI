# Требования к проекту To-Do List

## Основные требования

1. **Функциональность**:
   - Создание задач (to-do items)
   - Просмотр списка всех задач
   - Обновление статуса задач (выполнена/не выполнена)
   - Удаление задач
   - Редактирование текста задач

2. **Технические требования**:
   - Язык программирования: Python
   - База данных: SQLite
   - Без авторизации
   - Минимальный рабочий проект

3. **Дополнительные требования**:
   - Возможность написания тестов
   - Чистая архитектура для легкого тестирования
   - Простой и понятный код

## Нефункциональные требования

1. **Производительность**:
   - Быстрый отклик на запросы
   - Эффективная работа с базой данных

2. **Качество кода**:
   - Читаемый и поддерживаемый код
   - Соблюдение принципов SOLID
   - Хорошо структурированный проект

3. **Тестируемость**:
   - Возможность написания unit-тестов
   - Возможность написания интеграционных тестов
   - Четкое разделение логики и данных
